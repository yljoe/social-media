# Design Rules

## 產品定位

這個產品的 UI 不是娛樂型、不是行銷型，也不是泛用 SaaS 模板。它是：

- 企業情報控制台
- 社交工程內容生成工作台
- 管理後台 + 預覽編輯台的混合體

因此視覺上應該：

- 可信
- 冷靜
- 清楚
- 高資訊密度
- 以操作效率優先

## 字體系統

### 主字體

```css
font-family: "Noto Sans TC", "PingFang TC", "Microsoft JhengHei", sans-serif;
```

用途：

- 導覽
- 標題
- 內文
- 表單
- 表格

### 數據字體

```css
font-family: "Inter", "Noto Sans TC", "Segoe UI", sans-serif;
```

用途：

- KPI
- 編號
- 狀態數字
- 成本
- 時間

### 等寬字體

```css
font-family: "JetBrains Mono", "SFMono-Regular", Consolas, monospace;
```

用途：

- JSON
- token
- provider path
- job id

## 色彩方向

- 主背景偏淺灰或淺冷白
- 側欄用深色，建立穩定感
- 主 CTA 使用低彩度深藍、石墨藍、冷鋼藍
- Badge 顏色要節制：
  - active: 藍
  - review: 灰藍
  - completed: 淺藍灰或綠灰
  - error: 紅

不要：

- 大面積高飽和紫色
- 過量霓虹光
- 太多彩度相近的藍灰，導致層級分不清

## 版型規則

### 全域骨架

- 左側導覽固定
- 上方可有頁面說明或全域工具列
- 主要內容區寬度穩定
- 預覽與設定區維持成對出現

### 區塊優先級

1. 頁面標題與任務說明
2. 主工作區
3. 關鍵列表或預覽
4. 次級摘要
5. 洞察與提示

不要把第 4、5 層資訊放得比主工作區更顯眼。

## 列表設計規則

這是本 skill 最重要的部分。

### 目標

讓使用者在 3 秒內看懂：

- 這列是什麼
- 現在狀態是什麼
- 來自哪裡
- 最近什麼時候更新
- 下一步可以做什麼

### 列表格式

- 首欄：主名稱 + 次要 ID
- 第二欄：狀態 badge
- 第三欄：Provider / 分類 / 來源
- 第四欄：更新時間
- 第五欄：動作

### 行高與密度

- 每列資訊 2 層內解決
- 不要一列塞 4 行以上
- 主要資訊字重較高
- 次要資訊用較低對比

### 動作

- 預設顯示主操作，不要 hover 才出現
- 次要操作可收進更多選單
- 危險操作需二次確認

## 生成頁規則

- 左側為輸入與參數
- 右側為內容預覽
- 預覽上方可用 tabs 切換輸出形態
- 洞察區放在設定區下方或預覽區上方
- 生成按鈕必須明確、突出、唯一

## 文案規則

- 預設繁體中文
- 名詞一致，不混用同義詞
- 按鈕文案用動詞
- section title 用 2 到 6 字的短語
- 英文只保留產品名、技術名、專有名詞

## 實作規則

- 使用語意化 HTML
- icon-only button 一律加 `aria-label`
- 表單一律有 label
- `focus-visible` 必須清楚
- 手機版不得橫向爆版
- 避免 `transition: all`
- 避免把重要資訊只靠顏色表達
