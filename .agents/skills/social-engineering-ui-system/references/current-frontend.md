# Current Frontend

當任務是修改此專案 UI 時先讀這份，不要假設這是空白專案。

## 技術基底

- 前端位於 `frontend/`
- 使用 React 19 + TypeScript + Vite
- 目前主要樣式在：
  - `frontend/src/App.css`
  - `frontend/src/index.css`
- 主殼在：
  - `frontend/src/App.tsx`

## 目前資訊架構

主頁目前已經有左側導覽與中央工作區，模組包含：

- `檔案生成`
- `影片生成`
- `檔案管理`
- `API 管理`
- `素材管理`
- `費用管理`
- `使用教學`

這代表新 UI 應優先延續「工作台 / 控制台」邏輯，而不是改成行銷網站結構。

## 目前可沿用的優點

- 有明確 sidebar
- 有 hero 區塊與當前專案資訊
- 有生成頁與管理頁的分頁概念
- 已有繁體中文文案基礎
- 已有 focus-visible 基礎樣式

## 目前需要持續改善的方向

- 列表與表格可再更清楚、更一致
- Hero 與卡片視覺語言需要再統一
- 模組之間的視覺密度還不夠一致
- 生成頁的設定與預覽邏輯應更模板化
- 管理頁應更明確區分搜尋、篩選、表格、編輯器

## UI 工作時的預設策略

- 優先改善現有架構，不先推翻
- 優先把頁面標準化成固定骨架
- 優先讓列表、表格、預覽區更強
