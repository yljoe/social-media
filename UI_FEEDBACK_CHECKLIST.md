# UI Feedback Checklist

這份清單定義此平台的互動回饋規則，避免所有成功訊息都跑到頁面最上方，讓使用者不知道哪個按鈕真的生效。

## 回饋分級

1. 卡內提示
- 用在「載入表單」「切換編輯模式」「儲存成功」「切換到該模組」。
- 顯示位置：卡片標題下方或主要 CTA 上方。
- 例子：`已載入「shared 複本」到工作設定檔表單。`

2. 列內提示
- 用在單筆測試、套用、下載、檔案操作。
- 顯示位置：該列按鈕下方或該列內。
- 例子：storage 連線測試成功、檔案已下載。

3. 全域提示
- 只保留給跨模組跳轉、批次刪除、重大錯誤、需要整頁注意的事件。
- 顯示位置：hero 下方。
- 例子：`影片任務已送出。`、`已刪除 3 個專案。`

## 本專案落地規則

- 專案管理
  - 建立/編輯/載入專案：顯示在專案表單卡內。
  - 建立/編輯/載入工作設定檔：顯示在工作設定檔卡內。
  - 刪除專案/設定檔：保留全域提示。

- 供應商管理
  - 切換到新增 storage、載入 system 副本、套用 storage、儲存供應商：顯示在供應商編輯卡內。
  - 測試結果：顯示在按下測試的列或編輯器內。

- 素材管理
  - 載入素材、匯入素材、儲存素材：顯示在素材編輯卡內。
  - 刪除素材：保留全域提示。

- 內容生成
  - 成功生成草稿、套用 JSON 到影片生成：顯示在內容生成卡內。

- 影片生成
  - 準備 render request：顯示在影片生成卡內。
  - 真正送出影片任務：保留全域提示，因為會跨模組跳到檔案管理。

## Sidebar 規則

- sidebar 視為獨立導覽 panel。
- 桌面版固定在左側，使用 `sticky + 100vh`。
- 右側內容可自由捲動，sidebar 不應跟著內容滑走。
- 只有手機版才退化成一般流式排版。
